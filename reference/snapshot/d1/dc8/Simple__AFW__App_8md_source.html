36 @section writing-an-afw-app Writing an Aurora application
37 
38 An Application is an app handled by Aurora to run code. You supply it the main method and arguments.
39 
40 There are different types of Applications. The simplest one (and the one we're using for the tutorial) comes from the Core module, and is simply an Application.
41 
42 A Graphical Application (from the GEngine module), for example, already creates a window and uses a GraphicsAPI backend, being suitable to immediately render graphics on the screen.
43 
44 For now, let's bootstrap our code:
45 
46 ```cpp
47 
48 #include <AuroraFW/Aurora.h>
49 
50 using namespace AuroraFW;
51 
52 Application *app;
53 
54 void mainAppFunction()
55 {
56 
57 }
58 
59 int main(int argc, char *argv[])
60 {
61  app = new Application(mainAppFunction, argc, argv);
62 
63  delete app;
64  return EXIT_SUCCESS;
65 }
66 ```
67 
68 If you compile and run this, nothing happens yet. However, you're already using Aurora's Application class. Let's go line by line:
69 
70 ```cpp
71 #include <AuroraFW/Aurora.h>
72 ```
73 This includes the Aurora.h to your program, which includes every header from the modules you're using.
74 
75 ```cpp
76 Application *app;
77 ```
78 Here we define a pointer to an Application so we can define it later in the `int main()`
79 
80 ```cpp
81 void mainAppFunction()
82 {
83 
84 }
85 ```
86 
87 This is the method that will be executed by the Application class. Here you implement the code you want.
88 
89 ```cpp
90 int main(int argc, int *argv[])
91 {
92  ...
93 }
94 ```
95 
96 This is the main method of your program. Here you should initialize Application and, in case you're using a pointer, delete it when it's done.
97 
98 @note The `int argc, int *argv[]` arguments are optional, but it's a good pratice to include them, since Aurora uses them for it's own arguments
99 
100 ```cpp
101 app = new Application(mainAppFunction, argc, argv);
102 ```
103 
104 This is where the magic happens. Here you tell Application which method is your main method.
105 
106 You can get more details by reading the documentation for AuroraFW::Application.
107 
108 After your method is over, the code returns to your main method. As such, you must clean-up (in case you're using pointers):
109 
110 ```cpp
111 delete app;
112 return EXIT_SUCCESS;
113 ```
114 
115 So this is the basic code you must write to use Aurora. But this does not print anything. Let's use the method CLI::Log from the cli module to actually print something.
116 
117 @section printing Printing something
118 
119 Here we will use the CLI::Log() method to print something to the console:
120 
121 ```cpp
122 CLI::Log(CLI::Information, "Hello world!");
123 ```
124 
125 And now the code shall look like this:
126 
127 ```cpp
128 
129 #include <AuroraFW/Aurora.h>
130 
131 using namespace AuroraFW;
132 
133 Application *app;
134 
135 void mainAppFunction()
136 {
137  CLI::Log(CLI::Information, "Hello world!");
138 }
139 
140 int main(int argc, char *argv[])
141 {
142  app = new Application(mainAppFunction, argc, argv);
143 
144  delete app;
145  return EXIT_SUCCESS;
146 }
147 ```
148
